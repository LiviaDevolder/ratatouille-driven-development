# Template de História de Usuário

Use este formato para todas as histórias produzidas.

```markdown
# [Título curto e descritivo]

## História de Usuário

Como **[papel/perfil do usuário]**,
eu quero **[ação/atividade]**,
para que **[benefício/razão de negócio]**.

## Critérios de Aceitação

- [ ] [Comportamento observável e testável — cenário principal]
- [ ] [Comportamento observável e testável — variação ou edge case]
- [ ] [Comportamento observável e testável — estado de erro]
- [ ] [Comportamento observável e testável — limites/restrições]

## Notas

- **Dependências:** [outras histórias ou sistemas que precisam estar prontos]
- **Fora do escopo:** [o que explicitamente NÃO faz parte desta história]
- **Contexto:** [informações adicionais relevantes para o time]
```

## Regras do Template

1. **Título** — Máximo 10 palavras, descreve a ação do ponto de vista do usuário.
2. **Papel** — Deve ser específico (não use "usuário" genérico). Ex: "operador de pátio", "gerente comercial".
3. **Ação** — Uma única ação clara. Se tem "e", separe em duas histórias.
4. **Benefício** — Valor real de negócio, não reafirmação da ação.
5. **Critérios** — Mínimo 3, máximo 8. Cada um é um checkbox verificável.
6. **Notas** — Opcional, mas inclua dependências se existirem.
